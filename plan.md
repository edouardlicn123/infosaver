# 客户信息管理系统 - 项目计划书

## 一、项目概述

基于 Flask + SQLite 的轻量级客户信息管理系统，用于收集、整理和导出客户联系资料。

## 二、技术架构

| 组件 | 技术选型 |
|------|----------|
| 后端框架 | Flask |
| 数据库 | SQLite |
| 前端 | HTML + CSS (Bootstrap) + JavaScript |
| 数据处理 | Python + pandas |

## 五、功能模块

### 1. 收集模块（核心：智能解析）

#### 1.1 数据预处理策略

| 异常类型 | 处理策略 |
|----------|----------|
| 重复内容 | 检测连续重复字符串并去重（如 "Colin WongColin Wong" → "Colin Wong"） |
| 多余空格 | 移除字段内多余空格，统一分隔符 |
| 换行符 | 将换行转换为统一分隔符 |
| 字段错位 | 识别后标记异常，供用户确认页修正 |

#### 1.2 智能解析流程

```
原始数据输入
     │
     ▼
┌─────────────┐
│  预处理    │ ──► 去除重复、去空格、标准化换行
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  字段识别   │ ──► 以邮箱为锚点定位，提取前后字段
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  智能拆分   │ ──► 姓名/职位 去重处理
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  确认页展示 │ ──► 用户可编辑修正
└─────────────┘
```

#### 1.3 解析算法设计

- **核心思路**: 以邮箱为锚点，邮箱前是姓名+职位，邮箱后是来源
- **姓名去重**: 检测 `str + str` 模式，去重后保留一份
- **职位去重**: 同上
- **无邮箱行处理**: 标记为解析失败，单独展示供用户手动处理
- **验证规则**: 邮箱格式验证

#### 1.4 确认页面

- 表格形式展示解析结果
- 每条记录字段可编辑
- 解析异常数据高亮显示
- 支持删除某条记录
- 支持新增/删除行
- 批量确认保存

### 2. 管理模块

- 列表页：分页、搜索（姓名/邮箱）、排序
- 编辑页：完整字段编辑
- 删除功能（带确认）

### 3. 导出模块

- XLS 格式（Excel）
- CSV 格式
- 可选：全部字段导出 或 仅邮箱导出

## 六、页面路由设计

| 路由 | 方法 | 功能 |
|------|------|------|
| `/` | GET | 首页，重定向到收集页 |
| `/collect` | GET | 收集输入页面 |
| `/collect/preview` | POST | 智能解析并预览 |
| `/collect/confirm` | POST | 保存到数据库 |
| `/customers` | GET | 客户列表（分页、搜索） |
| `/customers/<id>` | GET | 查看/编辑单条记录 |
| `/customers/<id>` | POST | 更新客户信息 |
| `/customers/<id>/delete` | POST | 删除客户 |
| `/export` | GET | 导出选择页面 |
| `/export/xls` | GET | 导出XLS |
| `/export/csv` | GET | 导出CSV |

## 七、建议

### 7.1 解析策略建议

| 优先级 | 建议 | 说明 |
|--------|------|------|
| P0 | 邮箱识别优先 | 保证邮箱提取准确，姓名/职位允许用户在确认页修正 |
| P1 | 智能去重 | 自动处理 "NameName" 类重复问题 |
| P2 | 异常高亮 | 解析失败的行单独标记，引导用户手动处理 |

### 7.2 数据质量建议

| 建议 | 说明 |
|------|------|
| 重复检测 | 保存前检查同名+同邮箱，避免重复录入 |
| 数据验证 | 邮箱格式校验，无效邮箱提示 |
| 操作反馈 | 成功/失败 toast 提示 |

### 7.3 扩展性预留

| 建议 | 说明 |
|------|------|
| 用户表预留 | 虽然当前不需要认证，但预留 users 表结构 |
| 备份功能 | 定期自动备份数据库文件 |
| 日志记录 | 记录导入/导出操作日志 |

## 八、开发顺序

1. **第一阶段**: 数据库模型 + 基本路由
2. **第二阶段**: 收集模块（输入 + 解析 + 确认页）
3. **第三阶段**: 管理模块（列表 + 编辑）
4. **第四阶段**: 导出模块
5. **第五阶段**: 优化与测试
